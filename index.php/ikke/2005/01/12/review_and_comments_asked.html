Comment from: Ikke [Member] · http://www.eikke.com
[email sent to Joe]

Hi,

Thanks for commenting. I just read your project page, and got some
thoughts.

As far as I understand, Frege is based around files, not applications,
like DAL would. So I think it's got a much better use inside Beagle
(report more information on Beagle search results), or could be very
helpful with Gnome Storage (something upcoming but IMHO very promising,
check http://www.gnome.org/~seth/storage/ ).

As related to DAL, I don't think it is very similar. DAL mostly passes
through "events". Not "information". There's a big difference between "a
file has been started playing", or "this is some information about that
file".

Naturally, a lot of applications taking use of DAL will be able to take
advantage of Frege: if DAL tells them "a new track has been started,
this is the path to the playing file", they can use Frege to get
information about this file, without being forced to know how to access
metadata of this file (mp3 ID3, ogg metadata,...). As long as that's
what Frede would do? (Just a small part of course). Getting artist
information from the web could also be possible in some unified way etc
etc etc. At least, that's what I understand about what Frege would do.

Maybe Frege could also provide IM buddy information etc?


Apart from this: I'd like DAL to become a freedesktop.org project,
because the functionality it provides can be used by all possible
desktop environments, not only Gnome OR KDE OR whatever.
Frege seems to be Gnome-specific (which isn't a bad thing of course, we
all know Gnome kicks ass). It wouldn't be written using Mono/Nemerle
either (I don't even "talk" that language), but just plain C, so it is
easy to bind to any imaginable IPC protocol/method/... (they almost all
got some C binding, isn't it?)


Although I still don't know 100% what Frege would be doing (hey, I
hardly know what RDF is ;-)), the image I got now is promising and most
certainly has got it's use, mostly in combination with other
file-centric technologies like Beagle, GnomeStorage, Dashboard (which is
Beagle-based),...

Keep it going, and good luck with your dissertation!!!
